Applying Probability Concepts and Distributions

Normal Distribution (Gaussian)

Caveman: Most data clusters around the middle, extremes are rare.

The bell curve is fundamental because:

  • 68-95-99.7 rule: ~68% within 1σ, ~95% within 2σ, ~99.7% within 3σ
  • Many natural phenomena follow this pattern
  • Central Limit Theorem makes it universally applicable
Key Distributions
  • Normal: Continuous, symmetric (heights, errors)
  • Binomial: Discrete, fixed trials (success/failure counts)
  • Poisson: Discrete, rare events (customer arrivals, defects)
  • Exponential: Time between events (waiting times)

1.3.2

Shaping Samples Using the Central Limit Theorem

The CLT Magic

Caveman: Take enough samples, their averages become normal—no matter what the original data looks like.

Formal statement: Given a sufficiently large sample size (n ≥ 30), the distribution of sample means approximates a normal distribution, regardless of the population's distribution.

Why It Matters
  • Enables parametric tests even with non-normal populations
  • Foundation for confidence intervals
  • Justifies using sample statistics to infer population parameters
Standard Error of the Mean

SEM = σ / √n

As sample size increases, SEM decreases → more precise estimates

1.3.3

Estimating Parameters with Confidence Intervals

What CI Means

Caveman: "I'm 95% confident the true value is somewhere in this range."

Formula: CI = x̄ ± (z × SEM)

For 95% CI: z = 1.96

Common Misinterpretation

WRONG: "There's a 95% probability the true mean is in this interval"

RIGHT: "If we repeated this sampling 100 times, ~95 of the intervals would contain the true mean"

import scipy.stats as stats
import numpy as np

# Calculate 95% confidence interval
sample_mean = np.mean(data)
sample_std = np.std(data, ddof=1)  # ddof=1 for sample std
n = len(data)
sem = sample_std / np.sqrt(n)

# Using t-distribution for small samples
confidence = 0.95
ci = stats.t.interval(confidence, df=n-1, loc=sample_mean, scale=sem)
print(f"95% CI: ({ci[0]:.2f}, {ci[1]:.2f})")

Exploring Hypotheses

P-Value: The Decision Maker

Caveman: How likely you'd see this result if nothing was actually happening.

Pearson vs Spearman

Caveman: Pearson measures straight-line relationships; Spearman measures "when one goes up, does the other tend to go up too?"

Pearson (r) Spearman (ρ)
Linear relationships Monotonic relationships
Continuous data Ordinal or non-normal data
Sensitive to outliers Robust to outliers

  • 0.3 ≤ |r| < 0.7: Moderate

Predicting with Regression: Coefficients, Intercepts, Residuals

Linear Regression Equation

y = β₀ + β₁x + ε

  • β₀ (intercept): Value of y when x = 0
  • β₁ (coefficient): Change in y for 1-unit increase in x
  • ε (residual): Prediction error (actual - predicted)
R² (Coefficient of Determination)

Caveman: What % of the outcome does your model explain?

  • R² = 0.80: Model explains 80% of variance
  • Higher R² = better fit (but beware overfitting)
  • Adjusted R²: Penalizes adding useless features
from sklearn.linear_model import LinearRegression
from sklearn.metrics import r2_score, mean_squared_error
import numpy as np

# Simple linear regression
model = LinearRegression()
model.fit(X, y)

# Coefficients and intercept
print(f"Intercept: {model.intercept_}")
print(f"Coefficient: {model.coef_}")

# Predictions and evaluation
y_pred = model.predict(X)
r2 = r2_score(y, y_pred)
rmse = np.sqrt(mean_squared_error(y, y_pred))

# Residuals
residuals = y - y_pred
2.3.2

Regression Assumptions (LINE)

LINE Assumptions
  • L - Linearity: Relationship between X and Y is linear
  • I - Independence: Observations are independent
  • N - Normality: Residuals are normally distributed
  • E - Equal variance (Homoscedasticity): Residual spread is constant
How to Check
  • Linearity: Scatter plot of X vs Y
  • Independence: Durbin-Watson test
  • Normality: Q-Q plot of residuals
  • Homoscedasticity: Residuals vs fitted values plot
2.3.3

Multiple Linear Regression

Multiple Predictors

y = β₀ + β₁x₁ + β₂x₂ + ... + βₙxₙ + ε

Each coefficient represents the effect of that variable holding all others constant.

Watch for Multicollinearity

When predictors are highly correlated with each other:

  • Coefficients become unstable
  • Can't interpret individual feature importance
  • Solution: Check VIF (Variance Inflation Factor)

4.1.7

VIF: Detecting Multicollinearity

What VIF Tells You

Shanup: "VIF only looks at multicollinearity. It does not care about predictability. It tells you if features are moving together, same direction or opposite direction."

Why Multicollinearity Matters

Shanup: "When there is multicollinearity, you cannot isolate the effect of each variable on the target. Your boss trusts you and doubles marketing spend expecting 10x revenue, but there's another correlated factor you didn't account for. That's the biggest damage."

VIF Thresholds
  • VIF = 1: No correlation
  • VIF < 5: Generally acceptable
  • VIF ≥ 5-10: Problematic
  • Remove one feature at a time, recalculate VIF
from statsmodels.stats.outliers_influence import variance_inflation_factor
import pandas as pd

def calculate_vif(df):
    vif_data = pd.DataFrame()
    vif_data["Feature"] = df.columns
    vif_data["VIF"] = [variance_inflation_factor(df.values, i)
                       for i in range(df.shape[1])]
    return vif_data.sort_values('VIF', ascending=False)

# Calculate VIF
vif_results = calculate_vif(X)
print(vif_results)

# Remove highest VIF feature iteratively until all VIF < 5

Principal Component Analysis (PCA)

What PCA Does

Caveman: Squashes many dimensions into fewer while keeping the important stuff.

PCA finds new axes (principal components) that capture maximum variance. PC1 captures most variance, PC2 captures next most, etc.

Shanup on PCA Tradeoffs
"Once you do PCA transformation, you lose all interpretability. The features become PC1, PC2, PC3. You cannot tell your boss 'increase PC1 to increase revenue.' But multicollinearity is gone—all components are orthogonal."
When to Use PCA
  • Reduce computation time (many features)
  • Handle multicollinearity
  • Visualize high-dimensional data in 2D/3D
  • When interpretability isn't critical
from sklearn.decomposition import PCA
from sklearn.preprocessing import StandardScaler
import matplotlib.pyplot as plt

# IMPORTANT: Scale before PCA (variance-sensitive)
scaler = StandardScaler()
X_scaled = scaler.fit_transform(X)

# Fit PCA
pca = PCA()
X_pca = pca.fit_transform(X_scaled)

# Explained variance ratio
print("Explained variance:", pca.explained_variance_ratio_)
print("Cumulative:", np.cumsum(pca.explained_variance_ratio_))

# Choose components that explain 95% variance
pca_95 = PCA(n_components=0.95)
X_reduced = pca_95.fit_transform(X_scaled)

Detecting Anomalies with Unsupervised Learning

5.1.1

Types of Anomalies

Three Types of Anomalies
  • Point anomaly: Single observation differs significantly (e.g., deposit 10x normal income)
  • Collective anomaly: Group of observations show unusual pattern (e.g., sequence of small purchases indicating fraud)
  • Contextual anomaly: Unusual in specific context (e.g., web traffic spike normal during sale, anomaly otherwise)
Business Applications
  • Finance: Fraud detection
  • Cybersecurity: Intrusion detection
  • Manufacturing: Equipment failure prediction
  • Healthcare: Abnormal patient readings
5.1.2

Anomaly Detection with Z-Score

Z-Score Formula

z = (x - μ) / σ

Measures how many standard deviations from the mean.

Threshold Selection
  • |z| > 3: Common threshold (0.3% probability)
  • |z| > 2: More sensitive (5% probability)
  • Requires: Normal distribution

Threshold depends on business context!

from scipy import stats
import numpy as np

# Calculate z-scores
z_scores = stats.zscore(data)

# Identify anomalies (threshold = 3)
threshold = 3
anomalies = np.abs(z_scores) > threshold

print(f"Found {anomalies.sum()} anomalies")

# Get anomaly indices
anomaly_indices = np.where(anomalies)[0]
5.1.3

Anomaly Detection with IQR

IQR Method

Caveman: Use the middle 50% of data to define what's "normal."

  • IQR = Q3 - Q1
  • Lower bound = Q1 - 1.5 × IQR
  • Upper bound = Q3 + 1.5 × IQR
  • Extreme outlier: Beyond 3 × IQR
When to Use IQR vs Z-Score
  • IQR: Non-normal data, skewed distributions
  • Z-Score: Normal distributions
import numpy as np

def detect_outliers_iqr(data, multiplier=1.5):
    Q1 = np.percentile(data, 25)
    Q3 = np.percentile(data, 75)
    IQR = Q3 - Q1

    lower_bound = Q1 - multiplier * IQR
    upper_bound = Q3 + multiplier * IQR

    outliers = (data < lower_bound) | (data > upper_bound)
    return outliers, lower_bound, upper_bound

outliers, lb, ub = detect_outliers_iqr(data)
print(f"Lower bound: {lb}, Upper bound: {ub}")
print(f"Outliers: {outliers.sum()}")
5.2.2

K-Means Clustering for Anomaly Detection

Approach

Points far from all cluster centroids are potential anomalies.

  1. Run k-means clustering
  2. Calculate distance from each point to nearest centroid
  3. Flag points with large distances as anomalies
Caution

Anomalies close to a centroid or forming their own small cluster may not be detected.

from sklearn.cluster import KMeans
from sklearn.preprocessing import StandardScaler
import numpy as np

# Scale data
scaler = StandardScaler()
X_scaled = scaler.fit_transform(X)

# Fit k-means
kmeans = KMeans(n_clusters=5, random_state=42)
kmeans.fit(X_scaled)

# Calculate distances to nearest centroid
distances = kmeans.transform(X_scaled).min(axis=1)

# Threshold: points beyond 95th percentile
threshold = np.percentile(distances, 95)
anomalies = distances > threshold
5.2.3

One-Class SVM

How It Works

Caveman: Draws a boundary around "normal" data; anything outside is anomalous.

One-Class SVM learns a decision boundary around normal data points in high-dimensional space using the kernel trick.

Key Parameter: nu

nu: Upper bound on fraction of training errors and lower bound on fraction of support vectors (0 to 1).

from sklearn.svm import OneClassSVM
from sklearn.preprocessing import StandardScaler

# Scale data
scaler = StandardScaler()
X_scaled = scaler.fit_transform(X)

# Fit One-Class SVM
ocsvm = OneClassSVM(kernel='rbf', nu=0.05)
ocsvm.fit(X_scaled)

# Predict: 1 = inlier, -1 = outlier
predictions = ocsvm.predict(X_scaled)
anomalies = predictions == -1

print(f"Anomalies detected: {anomalies.sum()}")
5.2.4

Isolation Forest

The Intuition

Caveman: Anomalies are easier to isolate—they need fewer random cuts.

Isolation Forest builds random trees that partition data. Anomalies are isolated quickly (shorter path length) because they're different from the majority.

Advantages
  • Works well with high-dimensional data
  • Fast and efficient
  • No distance calculations needed
  • Handles mixed feature types
from sklearn.ensemble import IsolationForest

# contamination: expected proportion of outliers
iso_forest = IsolationForest(contamination=0.05, random_state=42)
iso_forest.fit(X)

# Predict: 1 = inlier, -1 = outlier
predictions = iso_forest.predict(X)
anomalies = predictions == -1

# Anomaly scores (more negative = more anomalous)
scores = iso_forest.decision_function(X)

6.1.1

What Is Clustering?

Clustering Defined

Caveman: Group similar things together without being told what "similar" means.

"The Milky Way is nothing else but a mass of innumerable stars planted together in clusters." — Galileo Galilei
Hard vs Soft Clustering
  • Hard: Each point belongs to exactly one cluster (k-means)
  • Soft: Points can belong to multiple clusters with probabilities
Business Applications
  • Customer segmentation: Target marketing
  • Market basket analysis: Product recommendations
  • Document clustering: Topic discovery
  • Image segmentation: Object detection
6.1.3

K-Means Algorithm

Algorithm Steps
  1. Choose k (number of clusters)
  2. Randomly initialize k centroids
  3. Assign each point to nearest centroid
  4. Recalculate centroids as mean of cluster
  5. Repeat 3-4 until convergence
Limitations
  • Must specify k in advance
  • Sensitive to initialization
  • Assumes spherical clusters
  • Sensitive to outliers
  • Only works with numerical data
from sklearn.cluster import KMeans
from sklearn.preprocessing import StandardScaler

# Scale data (important for k-means!)
scaler = StandardScaler()
X_scaled = scaler.fit_transform(X)

# Fit k-means
kmeans = KMeans(n_clusters=3, random_state=42, n_init=10)
kmeans.fit(X_scaled)

# Get cluster labels
labels = kmeans.labels_

# Cluster centroids
centroids = kmeans.cluster_centers_

# Inertia (within-cluster sum of squares)
print(f"Inertia: {kmeans.inertia_}")
6.1.4

Choosing K: Elbow and Silhouette Methods

Elbow Method

Plot inertia (WCSS) vs k. Look for the "elbow" where adding more clusters doesn't significantly reduce inertia.

Silhouette Score

Measures how similar points are to their own cluster vs other clusters.

  • Range: -1 to 1
  • 1: Perfect clustering
  • 0: Overlapping clusters
  • -1: Wrong cluster assignment
from sklearn.cluster import KMeans
from sklearn.metrics import silhouette_score
import matplotlib.pyplot as plt

# Elbow method
inertias = []
silhouettes = []
K_range = range(2, 11)

for k in K_range:
    kmeans = KMeans(n_clusters=k, random_state=42)
    kmeans.fit(X_scaled)
    inertias.append(kmeans.inertia_)
    silhouettes.append(silhouette_score(X_scaled, kmeans.labels_))

# Plot elbow
fig, axes = plt.subplots(1, 2, figsize=(12, 4))
axes[0].plot(K_range, inertias, 'bo-')
axes[0].set_xlabel('k')
axes[0].set_ylabel('Inertia')
axes[0].set_title('Elbow Method')

axes[1].plot(K_range, silhouettes, 'ro-')
axes[1].set_xlabel('k')
axes[1].set_ylabel('Silhouette Score')
axes[1].set_title('Silhouette Method')
plt.show()
6.2.1

Hierarchical Clustering & Dendrograms

Agglomerative (Bottom-Up)
  1. Start with each point as its own cluster
  2. Merge closest clusters
  3. Repeat until single cluster or desired number
Reading Dendrograms
  • Vertical lines: Cluster merges
  • Height: Distance at merge
  • Cut at height: Determines number of clusters
  • Long vertical lines: Natural cluster separation
from scipy.cluster.hierarchy import dendrogram, linkage, fcluster
from sklearn.cluster import AgglomerativeClustering
import matplotlib.pyplot as plt

# Create linkage matrix
linkage_matrix = linkage(X_scaled, method='ward')

# Plot dendrogram
plt.figure(figsize=(12, 6))
dendrogram(linkage_matrix)
plt.title('Hierarchical Clustering Dendrogram')
plt.xlabel('Sample Index')
plt.ylabel('Distance')
plt.show()

# Cut dendrogram to get clusters
clusters = fcluster(linkage_matrix, t=3, criterion='maxclust')

# Or use sklearn
hc = AgglomerativeClustering(n_clusters=3, linkage='ward')
labels = hc.fit_predict(X_scaled)
6.2.2

Linkage Methods: Ward, Complete, Single, Average

Linkage Comparison
Method Description Best For
Ward Minimize within-cluster variance Compact, equal-size clusters
Complete Maximum distance between clusters Tight clusters, sensitive to outliers
Single Minimum distance between clusters Elongated/chained clusters
Average Mean distance between clusters Balanced approach
6.2.3

Cluster Evaluation Methods

Internal Metrics (No Ground Truth)
  • Silhouette Score: Cohesion vs separation (-1 to 1)
  • Davies-Bouldin Index: Average similarity (lower = better)
  • Calinski-Harabasz Index: Variance ratio (higher = better)
External Metrics (With Ground Truth)
  • Adjusted Rand Index: Similarity to true labels (-1 to 1)
  • Normalized Mutual Information: Information shared (0 to 1)
from sklearn.metrics import (silhouette_score,
                             davies_bouldin_score,
                             calinski_harabasz_score)

# Internal metrics
silhouette = silhouette_score(X_scaled, labels)
db_score = davies_bouldin_score(X_scaled, labels)
ch_score = calinski_harabasz_score(X_scaled, labels)

print(f"Silhouette: {silhouette:.3f} (higher is better)")
print(f"Davies-Bouldin: {db_score:.3f} (lower is better)")
print(f"Calinski-Harabasz: {ch_score:.3f} (higher is better)")
